Arduino

Tijdens deze workshop zijn we voor het eerst aan de slag gegaan met de HCI kit. Hiervoor zijn we met het programma Arduino gaan programmeren. Hieronder zijn de volgende opdrachten uitgewerkt: 1, 2a, 2b, 2c, 2d, 3a, 3b, 3c, 4a, 4b, 4c, 5, 6a, 6b, 6c, 7 & 8. Tevens zijn er tevens video’s toegevoegd om te resultaat te laten zien.

Opdracht 3C

Bij de opdracht 3c was de uitwerking dat het groene lampje sneller hoorde te knipperen. Toen ik dit voor de eerste ging testen zag ik weinig verschil, ik heb hiervoor de code nog eens goed nagekeken en gekeken of alles goed was aangesloten op het breadboard. Dit was wel het geval, waarna ik hulp heb gevraagd aan klasgenoten en gevraagd of zij iets opvallends zagen. Ook de lange zijde van het lampje zat op de juiste plaats, omdat de zon buiten scheen dacht ik dat hierdoor het verschil lastig te zien was. Toen ben ik daarna een rondje gaan lopen of andere mensen dit probleem ook hadden. Hier kwam ik erachter dat het aan het lampje zelf lag. Ik heb hiervoor het lampje vervangen voor een rode(zoals te zien is in het voorbeeld). Dit lampje brandt feller en hierna heb ik de code opnieuw getest. Ik had per ongeluk beiden lampjes omgedraaid waardoor het rode lampje sneller knipperd in vergelijking met het gele lampje. Hiervoor moest wat worden aangepast in de void loop() om dit tot werking te laten komen.

Opdracht 4C

De volgende opdracht had als aanvulling op opdracht 3C, hiervoor moesten de twee lichtjes laden vervagen. Op het filmpje is te zien hoe de twee lichten beiden vervagen. Dit heb ik gerealiseerd door de brightness aan te passen in de loop middels < > deze tekens. In het begin lukte niet erg, dit komt mede doordat doordat de draadeje niet goed zaten bevestigd in het breadboard. Ook kwam ik een aantal draadjes tegen die niet goed genoeg connectie maakten. Toen ik dit eenmaal had aangepast en de code opnieuw had geüpload en deze getest. Ik ben zelf niet zo goed in coderen maar dooor deze opdrachten begin ik het steeds meer te begrijpen. Door de loop en elke keer het testen en doordat de opdrachten in etappes zijn is het leuk om te doen. Daarnaast zie je gelijk het resultaat wanneer iets is gelukt, voor het resultaat van opdracht 4c is hieronder te zien in het filmpje.

Opdracht 6C

De opdracht die daarop volgde heb ik de twee lampjes moeten laten vervagen middels Potentiometer. Wanneer er aan de potentiometer wordt gedraaid moeten beiden lichtjes vervagen. In het onderstaande filmpje is te zien wanneer ik aan de meter draai beiden lampjes vervagen. Ik had zelf nog nooit gehoord van deze meter, door elke keer een andere kant op te bewegen veranders het led-lampje.Ik vond het leuk om met de potentiometer te werken. Toen ik bij deze opdracht ermee voor het eerst in aanraking kwam had ik gelijk allemaal ideeën hoe ik deze kon inzetten. Zo kan de Potentiometer dienen als een button om iets aan en uit te zetten. Het lijkt mij leuk om de Potentiometer te gebruiken tijdens de hci weken.

Opdracht 7

Opdracht 7 ging het erom wanneer de lichtsensor werd aangeraakt of wanneer er schaduw was de led-lampjes gingen vervagen. Na het bouwen van het breadboard incl lichtsensor heb ik deze getest, in het filmpje hieronder is het resultaat te zien. Zoals te zien is heb ik na de workshop verder met de arduino kit gewerkt. Deze opdracht heb ik uitgevoerd in de Bibliotheek van Den Haag. Deze opdracht ging gelijk goed, zo had ik de snoeren die het niet meer goed deden apart gehouden. Wanneer ik ik mijn hand boven de sensor hield deed deze precies zoals deze het moest doen. Maar wanneer ik in de sensor zacht kneep zag je nog beter het verschil.

Opdracht 8B

Bij opdracht 8B heb ik deze gemaakt weer in de workshop les, hier gingen we verder met beiden programma’s. Hier hebben we geleerd hoe beiden programma’s samen werken en hoe je middels de code in Arduino en met de lichtsensor de een blokje kon laten bewegen. Door rect(float(sensorReading), 200, 200, 100); met nadruk op de cijfers pas je de beweging aan van het blokje. In het begin smapte ik het doel van de opdracht niet erg en wat er nu precies de bedoeling was. Nadat ik wat uitleg van crhis had gekregen snapte ik de opdracht en heb ik de code aangepast. Allereerst heb ik de vorm veranders, eerst was er een standaard voorbeeld maar je bij deze opdracht moest je het zo eigen mogelijk maken. Hiervoor heb ik de kleur en vorm van het voorwerp aangepast in de code. Hierna heb ik zoals hierboven beschreven.

Opdracht 9

Bij deze opdracht ging ik aan de slag met een buttons. Deze bevestig je op het breadboard van je arduino kit waarmee je met de onderstaande code het licht kan laten branden. Wanneer 1 knopje werd ingedrukt ging het led-lampje branden, wanneer beiden knopjes worden ingedrukt gaat het led-lampje weer uit. Bij deze opdracht werd er ook van ons gevraagd om de code te verkleinen, samen met klasgenoten hebben we gekeken hoe we deze zo klein mogelijk konden maken zonder dat de code stuk zou gaan. Tijdens de workshop kwamen we er niet echt uit en hebben we hulp gevraagd aan Chris Heydra. Hij heeft ons verder geholpen om de code te verkleine waarna bij de loop de code konden verkleinen naar 1 regels zelfs. Ik weet niet of ik hier ooit zelf op was gekomen maar na uitleg van chris heb ik er wel veel van geleerd om codes misschien te uitgebreid te schrijven.

Opdracht 10B

Bij deze opdracht ben ik aan de slag gegaan met de servomotoren. Hierbij moesten we een grappig ritme laten horen die de motor moest maken. Ik heb tijdens dit vergeten een opzetstuk erop te doen waarbij het dan duidelijker te zien zou zijn welke kant de motor op beweegt. Ik vond dit een super grappige opdracht en wist in eerste instantie niet zo goed wat ik kon verwachten van de servo motoren. Allereerst heb ik geprobeerd deze normaal te laten draaien. Hierna heb ik for loop een aantal keer herhaald en getallen aangepast zodat deze tijdens de loop elke wat anders doet.

Opdracht 11B

De opdracht die daarop volgde ging dieper in op het besturen van de servo motoren. Middels buttons die op het breadboard bevestigd waren was hiervoor de bedoeling dat deze door de buttons elke keer een andere kant op bewegen. Deze opdracht ging in combinatie met twee buttons, deze worden aangestuurd door de verschillende pins. Door deze te verwerken in de code samen met de servo aan te roepen in de while loop en aan te passen doet deze het zoals in de opdracht beschreven.

Opdracht 12B

Opdracht 12 bij ging ik aan de slag met de speaker die bij de kit zat geleverd. Middels de handleiding kon je verschillende noten in de loop zetten en wanneer de code runt komt er geluid uit de speaker. Door het invoeren van verschillende noten(codes) en deze achter elkaar zet in de loop ontstaat er een deuntje. In de video hieronder is te horen wat mijn ‘meesterwerk’ is geworden. Ik vond dit echt een super leuke opdracht om mee aande slag te gaan. In de studiehandleiding heb ik de noten die in de code gezet konden worden gehaald, zelf ben ik daarna nog opzoek gegaan op internet of en nog andere deuntjes beschikbaar waren. Hierna ben ik vervolgens op de Super Mario deuntje gekomen, Ik heb geprobeerd deze in de code te doen en werkend proberen te krijgen. Helaas is dit niet gelukt en heb ik het bij het onderstaande filmpje gehouden.

Opdracht 13E

Tot slot volgt opdracht 13E, Bij deze opdracht wordt gebruik gemaakt van de accelerometer. Deze meet de bewegingen van het breadboard. Door bewegingen te maken (board kantelen) komt er elke keer opnieuw een ander geluid uit de speaker. Wanneer het breadboard ongeveer 2 graden wordt gekanteld komt er geluid uit de speaker. In het begin kreeg ik de code maar niet werkend, wat ik ook deed en het breadboard bewoog en kwam geen geluid uit. Na vele malen proberen is het uiteindelijk gelukt. Hieronder is het resultaat te zien.
